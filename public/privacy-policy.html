Privacy Policy

Last updated: March 12, 2026

🔒 Privacy-First Guarantee: We do NOT store your email addresses on our servers. All validation results stay in YOUR browser only.

1. What Data We Collect

For DNS Validation (Default):

  • Domain names only (e.g., example.com)
  • We do NOT receive or see full email addresses
  • Email addresses are extracted in your browser, and only the domain is sent to our server

For SMTP Mailbox Verification (Optional - Deep Validation):

  • Full email addresses (e.g., user@example.com)
  • Required only for checking if mailbox actually exists
  • Used ONLY when you explicitly choose "Deep" validation level
  • Limited to 20 checks per minute to prevent abuse

What We Do NOT Collect:

  • ❌ No cookies
  • ❌ No analytics or tracking
  • ❌ No IP address logging
  • ❌ No user accounts or registration
  • ❌ No advertising or marketing data

2. How We Use Your Data

Zero Storage Architecture
Email addresses and domain names are processed in-memory only to perform validation checks. NO data is written to any database, file, or permanent storage.

Processing steps:

  1. You enter an email address in your browser
  2. Browser extracts domain name (e.g., example.com) for DNS checks
  3. Our server performs DNS lookup (MX, SPF, DKIM, DMARC)
  4. For SMTP verification (optional), full email sent to check mailbox existence
  5. Results returned to your browser
  6. Server immediately discards from memory (Node.js garbage collection)
  7. Results stored ONLY in your browser's localStorage

What this means:

  • ✅ We process your data to provide validation service
  • ✅ We do NOT store, log, or retain your email addresses
  • ✅ We do NOT use your data for any other purpose
  • ✅ We do NOT share your data with third parties

3. Data Storage Location

Your Browser (Client-Side):

  • All validation results are stored in your browser's localStorage
  • Stored on YOUR device only
  • We do NOT have access to this data
  • You can view it in: Browser DevTools → Application → Local Storage
  • You can clear it anytime using the "Clear All Data" button

Our Server (Temporary Processing Only):

  • ⚡ Email/domain processed for milliseconds only
  • 🗑️ Immediately discarded from memory after response
  • ❌ NO database, NO files, NO permanent storage
  • ❌ NO email addresses in server logs

4. Third-Party Infrastructure

Hosting Provider: Render.com

  • Security: SOC 2 Type II compliant
  • Encryption: HTTPS/TLS enforced for all connections
  • Logs: Basic HTTP request metadata (timestamps, status codes)
  • Data: Render does NOT log POST request bodies (where emails would be)
  • Retention: Server logs auto-deleted after 7-30 days
  • Location: United States (AWS infrastructure)

What Render Can See:

  • ✅ HTTP request metadata (timestamp, endpoint, status code)
  • ✅ Domain names in DNS lookups (not sensitive)
  • ❌ NOT email addresses (sent in encrypted POST body)
  • ❌ NOT your validation results (stored in your browser)

5. Your Data Rights

Right to Access:

View all your data stored in your browser:

  1. Open browser DevTools (F12 or Cmd+Option+I)
  2. Go to Application → Local Storage
  3. See all your validation results

Right to Delete:

Delete all your data:

  • Option 1: Click "Clear All Data" button in Settings tab
  • Option 2: Clear your browser cache/cookies
  • Option 3: Use browser's "Clear site data" feature

Right to Export:

Export your validation results:

  • Click "Export Results" button to download CSV file
  • Choose specific categories to export (valid, risky, invalid)

Right to Opt-Out:

  • SMTP verification: Optional - only happens if you choose "Deep" validation
  • DNS checks: Core functionality - required for validation service

6. Data Security

We implement industry-standard security measures:

  • 🔒 HTTPS/TLS encryption for all data in transit
  • 🔑 API key authentication to prevent unauthorized access
  • ⏱️ Rate limiting to prevent abuse (100 requests/min general, 20/min SMTP)
  • 🚫 No data persistence = no data breaches possible
  • ✅ Server-side validation to reject email addresses on DNS endpoints
